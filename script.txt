발표할 때 말할 부분
- Press the key 부분에 _getch함수를 써서 키 입력을 받아야 다른 동적으로 넘어간다.
- 벽에 닿으면 죽는 기능
- 자기 자신에 닿으면 죽는 기능
- 죽으면 생명이 줄어듦
- 별을 먹으면 점수가 올라감
- 실행 도중 아무키나 누르면 게임이 일시정지됨
- 일시 정지 상태에서 다시 방향키를 누르면 게임이 진행됨
- 게임 도중 esc를 누르면 게임이 종료됨
- 이스터 에그
	-뱀의 꼬리와 머리가 만나면 히든 필드가 나타남

- 배열을 이용하여 맵을 출력
- 구조체를 사용하여 뱀의 꼬리위치, 머리위치 몸통들의 위치를 저장
- 다른 구조체를 사용하여 각 몸통들이 이동할 방향과 현재 뱀의 머리가 이동할 방향 저장

- 처음에 맵 출려과 마찬가지로 뱀 몸통을 출력하는 것도 배열을 사용했었는데 배열에 뱀이 출력될 위치를 저장해두고 그 배열을 다른 함수에서 검사해서 화면상에 출력하는 방식이었다.
- 배열을 사용하게 되면 위치 저장, 출력과 같은 단계를 2번 거치게 되고 뱀이 출력되는 부분 외에도 for문이 공백이 저장되어있는 메모리들을 거쳐서 속도가 느려지고 다른 함수들을 연결했을 때 오류가 많이 발생했다.
- 그래서 다시 처음부터 코드를 작성해서 배열은 사용하지 않고 뱀의 위치는 저장하되 이스터에그를 구현하거나 음식을 먹을 때 벽에 부딫힐때만 사용했고, 뱀을 출력할 때는 gotoxy()함수를 이용하여 꼬리와 머리 위치만 저장해두고 위치가 저장된 다음에 바로 뱀을 출력하는 방식으로 코드를 짰습니다.

- 코드 줄은 대략 420줄 정도 나온것 같습니다.
